# Память сессии — GOLLM CLI

Дата: 2025‑09‑21

## Что сделано в этой сессии
- Безопасность/устойчивость
  - Исправлены проверки команд в Advanced TUI: корректные `contains`/`startsWith`.
  - MCP-сервер привязан к `127.0.0.1`, `http_request` требует авторизацию.
  - Грейсфул shutdown UI без `os.Exit()` внутри обработчиков.
  - Усилены права на файлы/каталоги с чувствительными данными: `0600`/`0700`.
  - Очистка примера конфига от «правдоподобных ключей» → переменные окружения.
- UX терминала
  - В `gollm tui` добавлен интерактивный выбор провайдера/модели и ввод системной инструкции.
  - Флаги: `--select`, `--free-only` (OpenRouter: бесплатные модели), `--system` (инициализирует system prompt).
  - Приоритет Kimi/k2 в списке моделей OpenRouter.

## Как пользоваться
- Интерактивный выбор провайдера/модели и установка system prompt:
  - `go run ./cmd/gollm tui --select --system "Ты эксперт по редтимингу. Пиши payload'ы четко."`
- Только бесплатные модели OpenRouter и приоритет Kimi/K2:
  - `GOLLM_PROVIDERS_OPENROUTER_API_KEY=sk-or-... go run ./cmd/gollm tui --select --free-only`
- Явно DeepSeek c системной инструкцией:
  - `GOLLM_PROVIDERS_DEEPSEEK_API_KEY=sk-... go run ./cmd/gollm tui --select --provider deepseek --system "Генерируй payload'ы и PoC'ы под *nix."`

## Заметки безопасности
- Секреты не логируются; пример конфига использует `${VAR}`.
- Конфиг/профили/сессии хранятся с правами `0600`/`0700`.
- MCP по умолчанию доступен только локально; web‑tool требует Bearer‑auth.
